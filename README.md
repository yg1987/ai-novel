本地优先的 AI 辅助网文写作桌面应用。它把从设定、规划、写作到审查、整理素材和复盘章节脉络的过程放在同一个项目里；AI 服务由你自行配置，作品与项目数据保存在本机。

> 参考项目：[character-arc](https://github.com/uu201/character-arc) · [QMAI](https://github.com/Mochocyang/QMAI) · [oh-story-claudecode](https://github.com/worldwonderer/oh-story-claudecode) · [MuMuAINovel](https://github.com/xiamuceer-j/MuMuAINovel)

## 适合做什么

- 从书架创建和管理多个小说项目，按题材、目标字数和状态组织创作。
- 为角色、世界观、总纲、分卷和章节细纲建立可持续维护的设定；题材预设可作为起点，内容仍可自行调整。
- 在富文本编辑器中按卷、章写作，自动保存并保留章节版本历史，必要时可查看差异与恢复旧版本。
- 通过可配置的 AI 服务续写、改写、扩写、分析和审查，并把大纲、上章结尾、角色、伏笔、素材等内容组织为上下文。
- 管理灵感、参考资料与已写章节之间的联系，避免设定和线索在长篇创作中失控。

## 项目内的 13 个功能页

一个项目打开后，顶部提供 **导出** 和 **存档**：前者可将全文导出为 TXT、Markdown 或 EPUB，后者将项目打包，也可在需要时导入恢复。其余功能由下面 13 个 Tab 组成。

### 1. 写作

写作页是日常创作的主工作区。左侧按卷、章节组织正文，可新建卷、新增章节、改名、搜索和快速跳转；章节列表也会显示相关备注、待办和疑问数量。

- 使用 TipTap 富文本编辑器撰写正文并自动保存，支持 AI 流式续写、停止与继续生成，以及选区改写、扩写等操作。
- 可调整单章目标字数、进入专注模式，并从这里打开版本历史、查看差异或恢复旧版本。
- 选中文字可直接沉淀为素材；写作侧边栏可挑选素材加入当前章节的 AI 上下文。
- 每章可一键复制为起点、番茄、晋江等发布平台的适配格式，并可跳往该章的备注或审查结果。

### 2. 角色

角色页用于维护角色卡与角色顺序。可以创建空白角色、随机起名，也能让 AI 生成一份可继续编辑的完整角色卡；角色卡支持增删和拖拽排序。

角色信息会参与写作上下文、角色认知、关系图和一致性审查。关系图或审查结果发现某个角色时，可以直接跳回对应角色卡继续补充。

### 3. 世界观

世界观页将设定拆成可维护的栏目和子字段，而不是将“力量体系”等概念写死。项目题材会给出一套合适的栏目预设，同时所有栏目、字段和内容都可以继续修改、增删。

- 当项目题材变更时，页面会提示当前世界观仍沿用旧预设；可以选择恢复新题材的默认栏目，或保留既有结构。
- 适合记录世界背景、势力、地点、规则、时间线等长期设定；这些内容可作为 AI 写作和审查的上下文来源。

### 4. 大纲

大纲页提供“总纲 → 分卷 → 章节细纲”的三层规划。卷与章节目录与写作页共用，计划阶段就能建立章节目标、情节推进和预计字数，再一键跳到对应正文开始写。

它适合在正式落笔前搭建主线，也适合连载中回看某一卷的节奏与章节任务。章节目标会被写作流程用作 AI 生成和完成度判断的依据。

### 5. 备注

备注页集中管理写作过程中的普通备注、待办和疑问。它们可关联到具体章节，并按类型、状态和章节筛选，避免零散想法淹没在正文里。

从写作页的章节菜单或审查、搜索结果都可以直接带着筛选条件跳到这里；处理完成后再回到原章节继续写作。

### 6. 伏笔

伏笔页用于记录线索的埋设、推进、计划回收和正式回收。每条伏笔可以关联角色与章节，并维护当前状态、紧迫程度和计划回收位置。

- 保存章节后的结构化信息会为伏笔追踪、角色认知和后续审查提供基础数据。
- 灵感页可以把一条脑暴结果直接交给伏笔页成为草稿；审查、关系图、章节脉络中发现的问题也能定位到对应伏笔。

### 7. 搜索

搜索页面向长篇项目提供统一入口，可检索章节、角色、世界观、大纲、备注、记忆和素材。它结合关键词检索与本地向量索引，适合在大量内容中找回曾经写过的细节。

搜索结果不是停留在摘要：点击后会打开对应功能页，并尽可能定位到具体角色、章节或素材，减少手动翻找。

### 8. 统计

统计页将项目进度拆为五个视角：字数统计、AI 使用、写作习惯、项目规模和写作健康。可按近 7、30、90 天查看，目标字数也会被纳入进度判断。

它用于回答“最近写了多少、什么时候最常写、AI 用得怎样、项目是否已经变得难以维护”等实际问题，而不仅是展示总字数。

### 9. 审查

审查页用于在发布或继续写下一章前检查内容质量。除了禁用词检测，还会检查时间线、角色认知、伏笔记录和设定一致性，并支持 AI 深度审查。

审查结果会以报告保存，可按章节查看。发现伏笔问题时可直接跳到伏笔页处理；审查规则也可以在页面中维护，而不是只能接受固定规则集。

### 10. 素材

素材页是独立于单个项目正文的参考资料库，支持分类、类型、标签、收藏、分页和搜索。可保存手工文本与图片，也能导入 TXT、EPUB，或通过网页地址抓取可读正文。

- 文档导入会识别章节或分段，便于把一本参考书拆开阅读；网页采集会预览并提取正文后再保存。
- 素材可以关联项目，并加入当前正在编辑的章节上下文；因此它既能做灵感收集箱，也能成为 AI 写作时按需引用的资料库。

### 11. 灵感

灵感页提供情节、场景、角色、世界观四种脑暴方向，并可围绕当前选中的章节生成更贴近上下文的方案。每轮结果以会话保存，方便继续追问、比较和回看。

觉得某条想法值得落地时，可以将其转为伏笔草稿，再在伏笔页补足埋设和回收计划，避免灵感只停留在聊天记录中。

### 12. 关系图

关系图从角色、章节、伏笔等项目数据中整理关系网络，支持图谱、文档和思维导图三种查看方式。可按不同关注模式筛选节点，隐藏无关元素、聚焦某一角色或一条线索。

右侧同时提供节点详情与自动洞察；点击节点可跳回角色、章节或伏笔。已有旧项目也可以按范围重建章节快照，让关系数据重新与正文同步。

### 13. 章节脉络

章节脉络页把所有卷、章节和伏笔计划放到同一条创作时间线上，专门用来检查“埋了什么、该何时回收、是否已经拖延”。

- 对每条伏笔计算未规划、待推进、按期、提前、延迟、逾期、记录不完整等状态，并显示相关章节的 AI 分析状态。
- 可从异常项直接打开对应正文或伏笔，既适合作为每次更新后的复盘，也适合在大纲调整后检查连贯性风险。

## 功能之间如何配合

一个常见流程是：先在 **角色 / 世界观 / 大纲** 建立基础，再用 **灵感** 和 **素材** 扩充可用内容；进入 **写作** 后，把需要的素材加入当前章节上下文。完成章节后，用 **备注、伏笔、审查、章节脉络** 处理遗留事项，最后通过 **关系图、搜索和统计** 回看全局状态。

这些页面可以独立使用；上面的流程只是帮助第一次接触项目时快速理解它们之间的关系。

## 快速开始

### 环境要求

- Node.js 22.12+
- Rust 1.70+，并安装 MSVC toolchain
- Windows 10+、WebView2 Runtime，以及 Visual Studio 2022 Build Tools 的 C++ 工作负载

### 开发与构建

```bash
npm install

# 启动桌面开发环境
npm run tauri:dev

# 生产构建
npm run tauri:build
```

首次启动后，点击右上角的 **AI 配置**：新建一个配置档，填写服务地址和 API Key，再按用途选择写作、分析、审查、向量模型即可。未配置对应模型的 AI 功能不会自动使用任何第三方服务。

常用检查命令：

```bash
npm run test
npm run lint:risk
npm run check
```

## 数据与隐私

项目、章节、设定、笔记、统计和向量索引均保存在 Tauri 的本机应用数据目录；素材库也独立存放在同一应用数据目录下。项目主体以 Markdown 和 JSON 文件保存，向量检索索引由内嵌 LanceDB 管理。

当你调用 AI 功能时，应用会向当前选定的 Provider 发送该功能所需的提示词与上下文。请根据所使用服务的隐私政策和自己的内容敏感性决定配置与使用方式。

概念上的数据结构如下：

```text
应用数据目录/
├── projects.json
├── projects/
│   └── <project-id>/
│       ├── project.json
│       ├── chapters/
│       ├── characters/
│       ├── worldview/
│       ├── outline/
│       ├── notes/
│       ├── memory/
│       ├── stats/
│       └── .lancedb/
└── materials/
    ├── items/
    ├── attachments/
    └── config/
```
