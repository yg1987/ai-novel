## 1. 快速开始

### 1.1 环境要求

- Windows 10+（需 WebView2 运行时，Windows 10 一般自带）
- Node.js 22.12+
- Rust 1.70+（仅开发构建时需要）

### 1.2 安装与启动

```bash
# 安装依赖
npm install

# 开发模式启动
npm run tauri:dev

# 生产构建（生成安装包）
npm run tauri:build
```

### 1.3 首次使用四步走

1. 启动应用 → 进入**书架**页面
2. 点击右上角 **⚙ AI 配置** → 填写你的 API 地址和 Key（支持 OpenAI / DeepSeek / Ollama 等）
3. 点击 **+ 新建项目** → 填写书名、类型、简介
4. 点击项目进入写作面板 → 选中左侧章节 → 点击 **🤖 生成本章**

## 3. 书架管理

书架是应用的主页，展示所有项目。

### 3.1 项目列表

- 每个项目显示为一张**卡片**，包含：书名、类型、状态、简介、目标字数
- 默认按**创建时间降序**排列（最新项目在最上面）
- 右侧操作按钮（hover 时浮现）：✎ 编辑、✕ 删除

### 3.2 项目排序

- 项目左侧显示状态标签颜色：
  - 🟠 连载中（橙色）
  - 🟢 完结（绿色）
  - ⚫ 搁置（灰色）

### 3.3 创建项目

点击 **+ 新建项目** 按钮，弹出对话框填写：

| 字段 | 说明 |
|------|------|
| 书名 | 必填，你的小说名称 |
| 类型 | 选填，如玄幻/都市/言情/科幻/悬疑/历史 |
| 状态 | 连载中 / 完结 / 搁置 |
| 简介 | 一句话简介（限 200 字） |
| 目标字数 | 全书目标总字数 |

### 3.4 编辑 / 删除项目

- **编辑**：hover 项目卡片 → 点击 ✎ 按钮 → 修改后保存
- **删除**：hover 项目卡片 → 点击 ✕ 按钮 → 弹窗确认 → 删除（**不可恢复**）

## 5. 写作面板

写作 Tab 是核心功能区，采用**左列表 + 右编辑器**布局。

### 5.1 章节列表（左侧侧栏）

- 显示当前项目的所有章节列表
- 点击章节名称切换编辑
- 章节按编号排序

### 5.2 编辑器（右侧）

基于 TipTap 的富文本编辑器，功能：

- **实时字数统计**：显示当前字数 / 目标字数
- **自动保存**：每次编辑后 3 秒自动保存到本地
- **阅读模式**：隐藏工具栏的纯净阅读布局
- **快捷键**：支持标准编辑快捷键

### 5.3 编辑器工具栏

编辑器上方工具栏提供：

- **粗体 / 斜体 / 下划线**：文字格式
- **撤销 / 重做**：编辑历史回退
- **🤖 生成本章**：调用 AI 生成全文
- **▶ 继续生成**：从光标位置继续 AI 生成
- **⏹ 停止**：中断 AI 生成
- **📖 阅读模式**：切换到纯阅读视图
- **专注模式**：锁定编辑器，屏蔽干扰

### 5.4 批注与 TODO

编辑器支持在正文中添加批注（不显示在最终文本中）：

- **写作备注**：给自己看的提示
- **TODO 标注**：待修改项标记
- **疑问标注**：需要确认的点

## 6. AI 生成正文

### 6.1 生成全文

1. 确保已在 ⚙ AI 配置 中填好 API 信息
2. 选择要生成的章节
3. 点击 **🤖 生成本章**
4. AI 流式输出内容到编辑器，实时渲染
5. 随时可点击 **⏹ 停止** 中断生成
6. 点击 **▶ 继续生成** 从光标位置继续

### 6.2 局部改写/扩写

1. 在编辑器中选中一段文字
2. 右键（或点击出现的浮动按钮）选择：
   - **AI 改写此段**：用 AI 重写选中内容
   - **AI 扩写**：扩展选中段落，增加细节

### 6.3 AI 上下文引擎

生成时 AI 会自动获取以下上下文（无需手动操作）：

- 当前章节细纲目标
- 上一章结尾内容
- 前 3 章摘要
- 本章出场角色的**认知状态**（知道/不知道/读者知道）
- 未解的**伏笔**（按紧迫度排序）
- 相关的设定知识（自动检索）

可在编辑器中打开 **显示上下文** 面板查看 AI 实际收到了哪些信息。

### 6.4 写后自动处理

章节保存后，系统自动执行：

1. **禁用词检测** → 标记疑似 AI 套话
2. **记忆提取** → AI 自动分析本章剧情，更新角色状态/伏笔/时间线
3. **向量索引更新** → 供后续语义搜索使用
4. **统计记录** → 字数、用时等写入日志

## 7. AI 配置

点击右上角 **⚙ AI 配置** 打开配置面板。

### 7.1 配置项

| 字段 | 说明 |
|------|------|
| 名称 | 给这个配置起个名字（如"主力"、"本地"） |
| API 地址 | OpenAI 兼容接口地址 |
| API Key | 你的 API 密钥 |
| 写作模型 | 用于正文生成的模型（推荐强模型） |
| 分析模型 | 用于章节分析/记忆提取的模型 |
| 审查模型 | 用于一致性审查的模型 |
| Embedding 模型 | 用于向量检索的嵌入模型 |

### 7.2 获取模型列表

填入 API 地址和 Key 后，点击 **🔍 获取模型列表**，系统会自动拉取可用模型列表，下拉选择即可。

### 7.3 支持的服务商

任何兼容 OpenAI API 的提供商，例如：

| 服务商 | API 地址示例 |
|--------|-------------|
| OpenAI | `https://api.openai.com/v1` |
| DeepSeek | `https://api.deepseek.com/v1` |
| Ollama（本地） | `http://localhost:11434/v1` |
| 其他兼容服务 | 填入对应地址即可 |

### 7.4 多 Provider 切换

可配置多个 Provider，在列表中切换激活配置。

## 10. 大纲系统

大纲 Tab 采用**三层大纲结构**，左列表 + 右侧详情布局。

### 10.1 三层结构

```
总纲（全书梗概）
  └─ 分卷（每卷的剧情走向）
       └─ 章节细纲（每章的情节点）
```

### 10.2 章节细纲

每个章节细纲包含：

- **情节点列表**：逐段描述本章发生的事
- **字数预算**：每段预计字数，总和不超过目标字数
- **段落类型**：铺垫 / 爽点 / 推进 / 高压 / 修炼 / 回收 / 生活 / 信息

### 10.3 操作

- 新建分卷/章节
- 拖拽调整顺序
- AI 生成大纲骨架：输入简介自动生成
- AI 章节扩展：选中细纲节点让 AI 填充情节点

## 12. 伏笔追踪

伏笔 Tab 是管理小说伏笔的专用面板。

### 12.1 伏笔状态

每个伏笔有以下状态机：

```
埋伏笔 → 待解 → 已解
               ↘ 废弃
```

### 12.2 伏笔属性

| 属性 | 说明 |
|------|------|
| 描述 | 伏笔内容 |
| 埋伏章节 | 在哪一章埋下的 |
| 目标解决章节 | 计划在哪一章回收 |
| 重要性 | 高 / 中 / 低 |
| 紧迫度 | 按当前进度自动计算（越接近目标越紧迫） |

### 12.3 自动追踪

系统会在每次章节保存后自动：

- 识别新埋伏笔
- 识别已解伏笔
- 更新紧迫度排序
- 生成**伏笔债评分**

## 13. 全局搜索

搜索 Tab 支持全文检索，采用**关键词 + 语义**混合搜索。

### 13.1 搜索范围

- 角色文件
- 世界观设定
- 章节正文
- 写作笔记
- 记忆数据
- 大纲

### 13.2 搜索方式

- **关键词搜索**：CJK 分词 + 文件名精确匹配
- **语义搜索**：向量检索，理解含义而非字面
- **混合搜索**：两种方式结果融合排序，结果更准确

### 13.3 搜索结果

结果按相关度排序，每条结果标注来源类型和匹配方式。

## 15. 内容审查

审查 Tab 提供写作质量保障功能。

### 15.1 AI 禁用词检测

基于词表自动检测 AI 写作常见"套话"：

- 翻转句式（"不是 A，而是 B"）
- 万能状语（"带着……"）
- 章末总结体
- 抽象升华词（命运、宿命）
- 万能比喻（潮水般、闪电般）
- 以及其他 25+ 模式

每种问题标注**1-5 星毒级**（5 星最严重）。

### 15.2 自动一致性审查

审查维度：

| 维度 | 说明 |
|------|------|
| 时间线 | 时间顺序是否一致，无矛盾 |
| 角色认知 | 角色不会知道不该知道的事 |
| 伏笔遗忘 | 检查是否有伏笔被遗忘 |
| 设定自洽 | 是否与世界观设定矛盾 |

审查结果分两级：

- **Tier 1**：确定性规则检查（无 AI 参与）
- **Tier 2**：AI 深度审查（调用 AI 分析）

### 15.3 操作

- **查看审查报告**：审查结果汇总
- **AI 自动修复**：小问题可让 AI 自动修正
- **手动重审**：修改后重新触发审查
- **版本历史**：查看章节修改历史

## 18. 关系图谱

关系图 Tab 展示角色之间的关系网络。

### 18.1 功能

- **力导向图**：基于 d3-force 的交互式可视化

## 23. 数据存储与备份

### 23.1 工作空间结构

应用数据全部存储在本地文件系统中：

```
ai-novel-workspace/
├── projects.json            # 书架索引
├── resources/               # 素材库（跨项目）
└── projects/
    └── <project-id>/
        ├── project.json     # 项目元数据
        ├── .lancedb/        # 向量索引（自动管理）
        ├── characters/      # 角色文件（.md）
        ├── worldview/       # 世界观设定（.md）
        ├── outline/         # 大纲
        ├── chapters/        # 章节正文（.md）
        │   └── .history/    # 章节版本历史
        ├── notes/           # 写作笔记
        ├── memory/          # AI 记忆数据（自动管理）
        ├── stats/           # 统计事件日志
        └── tracks/          # 审查报告
```

### 23.2 设计原则

- **本地优先**：所有数据在本地，不依赖云服务
- **文件即存储**：Markdown 正文可用任何文本编辑器直接打开
- **可迁移**：项目目录即为完整备份，打包即可带走

### 23.3 自动备份

系统维护循环备份策略，保留最近 N 个全项目快照。可在崩溃后一键恢复。

## 24. 常见问题

### Q1: 启动后报错 WebView2 未找到？

Windows 10 需安装 [Microsoft Edge WebView2 运行时](https://developer.microsoft.com/en-us/microsoft-edge/webview2/)。Windows 11 自带。如无，下载安装即可。

### Q2: AI 生成没反应？

1. 检查 ⚙ AI 配置 中 API 地址和 Key 是否正确
2. 点击 🔍 获取模型列表，确认模型可用
3. 确保网络连接正常（如使用云端 API）
4. 如使用本地 Ollama，确保 Ollama 服务已启动

### Q3: 如何更换 AI 模型？

在 ⚙ AI 配置 中编辑当前 Provider 的模型字段，或切换其他 Provider。

### Q4: 数据存在哪里？

工作空间默认在运行目录的 `ai-novel-workspace/` 下。所有数据均为本地文件，详见 [23. 数据存储与备份](#23-数据存储与备份)。

### Q5: 可以离线使用吗？

可以，前提是使用本地 AI 模型（如 Ollama + qwen2.5）。离线时云端 AI 功能不可用，但编辑和浏览本地数据完全正常。

### Q6: 如何把项目迁移到另一台电脑？

在旧电脑使用 **💾 存档** 按钮导出 tar.gz 文件，然后复制到新电脑，点击 **导入存档** 恢复。

### Q7: 生成的章节太"AI 味"怎么办？

写完一章后切到 **审查** Tab 运行禁用词检测，系统会标注 AI 常见的套话模式，可手动或让 AI 自动修复。

### Q8: 伏笔债评分低怎么办？

进入 **伏笔** Tab 查看按紧迫度排序的未解伏笔列表，优先回收紧迫度高的伏笔。
