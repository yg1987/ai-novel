# 统计模块设计说明 v2.1

> 场景定位：**写作仪表盘** — 看一眼就知道项目全貌。字数 = 文件快照（真实产出），趋势 = 事件日志（时间序列）。图表用 Recharts。

> **v2.1 更新**：写作日历从 GitHub 风格 heatmap 改为 CSS Grid 月历（占满宽度、大格子、显示日期+字数+AI 次数）；AI 次数使用文字 "AI: N" 而非 emoji；字数不缩写；有数据格子统一 accent 色；项目天数 `Math.max(1, ...)`。

---

## 一、核心设计决策

### 1.1 字数统计：只用文件快照

**不看事件累计，只看当前文件内容。** 遍历所有章节文件，`estimateWordCount` 求和 = 累计字数。

删了重写、改了内容，都以当前文件为准。因为"最终产出"才有统计意义——写了又删掉的 8 万字不是产出。

### 1.2 时间序列：保留 JSONL 事件日志

文件快照没有时间维度。以下数据必须依赖事件日志：
- 每日新增字数趋势（哪天写了多少）
- AI 生成次数 / Token 消耗
- 写作会话频率和时长
- 章数变化时间线

### 1.3 模块完全解耦

每个统计子模块（字数 / AI / 习惯 / 规模 / 健康）是独立的 React 组件，各自独立取数、独立渲染。`StatisticsPanel` 只做路由。子模块之间零依赖，不共享 state。

### 1.4 图表库：Recharts

React 原生声明式图表，tree-shakeable，按需引入 `BarChart` / `LineChart` / `PieChart` 等组件，不拖全量 D3。

## 二、数据架构

### 2.1 两种数据源，各司其职

| 数字类型 | 数据来源 | 说明 |
|---------|---------|------|
| 累计总字数 | 章节文件快照 | 遍历 `chapters/` 下所有 `.md` 文件，`estimateWordCount` 求和 |
| 每章字数 | 章节文件快照 | 逐章读取统计 |
| 每日新增字数 | JSONL 事件日志 | `chapter_saved` 事件 delta 聚合（已修复语义） |
| AI 使用数据 | JSONL 事件日志 | `ai_generated` 事件聚合 |
| 写作会话 | JSONL 事件日志 | `session_start` 事件计数 |
| 项目规模 | 文件系统 | `listProjectFiles` / `listChapters` 计数 |
| 伏笔健康度 | `foreshadows.json` | 已有 `calcForeshadowHealth` 逻辑复用 |
| 质量指标 | 审查报告文件 | `tracks/review-reports/` 下 JSON 读取 |

### 2.2 修复 StatEvent：delta 语义

当前 `logChapterSaved` 的 `word_count` 是整章总字数（每次保存重复计入）。改为 delta：

```typescript
// 改造后：只记录本次新增字数
const plainText = content.replace(/<[^>]*>/g, '')
const lastCharCount = lastChapterContent.get(key) ?? 0
const charDelta = plainText.length - lastCharCount
lastChapterContent.set(key, plainText.length)

if (charDelta <= 0 && lastCharCount > 0) return

const deltaWordCount = estimateWordCount(plainText.slice(lastCharCount))
const event = {
  timestamp: new Date().toISOString(),
  event_type: 'chapter_saved',
  chapter,
  char_count: charDelta,
  word_count: deltaWordCount,  // ← delta, 不再重复计入
  event_version: 1,            // ← 标记新语义
}
```

### 2.3 事件日志不做旧数据迁移

`event_version` 缺省为 None（旧事件），Rust 聚合时 None 的事件跳过 `word_count`，避免被旧的全量字数污染。当前都是测试数据，不需要迁移。

```
│   │   └── HealthSection.tsx      ← Phase C：健康度

src-tauri/src/commands/
└── stats.rs                   ← compute_project_stats + compute_chapter_word_counts
```

### 3.1 组件契约

每个 Section 组件接口一致：

```typescript
interface SectionProps {
  projectId: string
  days: number          // 时间窗口：7 / 30 / 90
}
```

Section 内部自行取数、自行处理 loading/empty/error 状态。`StatisticsPanel` 不向子组件传递数据，只传 `projectId` + `days`。

### 3.2 数据服务层

`statsService.ts` 提供纯计算函数，与 React 无关：

```typescript
// 文件快照
getChapterWordCounts(projectId): Promise<ChapterWordCount[]>
getTotalWords(projectId): Promise<number>

// 事件聚合（通过 Tauri 命令）
getDailyStats(projectId, days): Promise<DailyStats[]>
getAIUsageStats(projectId, days): Promise<AIUsageStats>
getSessionStats(projectId, days): Promise<SessionStats>

// 项目规模（文件计数）
getProjectScale(projectId): Promise<ProjectScale>

// 健康度（复用现有逻辑）
getForeshadowHealth(projectId): Promise<ForeshadowHealthMetrics>
getQualityTrends(projectId, days): Promise<QualityTrend[]>
```

## 五、后端修改

### 5.1 Rust 命令

```
// 主命令：一次返回所有统计数据
compute_project_stats(project_id: String, days: u32) → ProjectStats

// 辅助命令：章节字数快照
compute_chapter_word_counts(project_id: String) → Vec<ChapterWordCount>
```

### 5.2 数据结构

```rust
pub struct ProjectStats {
    // 字数
    pub total_words: u32,              // 文件快照求和
    pub total_chapter_words: u32,      // 同上，别名（语义清晰）
    pub avg_words_per_chapter: u32,
    pub max_chapter_words: u32,
    pub min_chapter_words: u32,

    // 章卷
    pub total_chapters: u32,
    pub total_volumes: u32,
    pub chapters_this_period: u32,     // days 窗口内新建的章数

    // AI
    pub total_ai_generations: u32,
    pub total_ai_tokens: u64,
    pub avg_ai_duration_ms: u64,

    // 会话
    pub total_sessions: u32,
    pub total_duration_ms: u64,
    pub writing_streak_days: u32,

    // 规模
    pub total_characters: u32,
    pub total_worldview_entries: u32,
    pub total_notes: u32,
    pub total_resources: u32,

    // 每日明细
    pub daily_stats: Vec<DailyStats>,
}

pub struct DailyStats {
    pub date: String,
    pub word_count: u32,        // delta 聚合
    pub char_count: u32,
    pub ai_generations: u32,
    pub ai_tokens: u64,
    pub sessions: u32,
    pub duration_ms: u64,
    pub chapters_created: u32,
}

pub struct ChapterWordCount {
    pub chapter_id: String,
    pub title: String,
    pub volume: String,
    pub order: u32,
    pub word_count: u32,
    pub char_count: u32,
}
```

### 5.3 `compute_project_stats` 数据整合逻辑

```
1. 读取 stats/*.jsonl → 按 event_version 过滤 + 按天聚合 → daily_stats
2. listChapters(project_id) → total_chapters, total_volumes
3. getProject(project_id) → 项目创建时间 → writing_streak_days
4. 遍历所有章节文件 → estimate_word_count → total_words, avg/max/min
5. listProjectFiles × 4 → total_characters/worldview/notes/resources
6. loadForeshadows → 复用 calcForeshadowHealth
7. 聚合 daily_stats 产生 total_ai_generations, total_ai_tokens 等
```
