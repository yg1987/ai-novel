# 关系图模块改造开发计划

## Goals

- 将现有 `RelationshipGraph` 从“角色-only + d3-force + 手写 SVG”升级为 Sigma.js v3 + graphology 多类型图谱。
- 支持 7 类节点：角色、地点、物品、组织、事件、章节、伏笔。
- 支持 5 种分析模式：总览、人物、章节、故事线、伏笔。
- 支持 3 种显示模式：力导向图、文档视图、脑图。
- 支持洞察分析：惊喜连接、孤立节点、稀疏社区、桥梁节点。
- 支持手动重新生成全部章节快照，失败不中断，完成后刷新关系图。
- 修复并防止 `node not found` 白屏：所有边进入 graphology 前必须检查端点存在。

## Constraints & Preferences

- 本轮只改造 `RelationshipGraph`，不迁移 `ChapterGraph`。
- 保留 `d3-force` / `@types/d3-force`，因为 `src/components/ChapterGraph.tsx` 仍依赖它们。
- 不修改快照 JSON 格式：`memory/snapshots/*.snapshot.json` 不变。
- 不改 Tauri Rust 后端。
- 不做复杂 NLP 组织抽取，只做保守规则解析。
- 不做跨章事件自动合并，事件 ID 使用 `event:${chapterId}:${index}`。
- `src/types/novel.ts` 只放领域数据类型；`GraphMode` / `GraphDisplayMode` / `GraphLabelVisibility` 只放 `src/lib/graph-mode.ts`。
- 业务服务层返回 plain data；graphology Graph 只作为运行时派生对象，不进 React state、不落盘。
- 代码质量检查必须使用 `lsp_diagnostics`，不得用 `tsc --noEmit` 替代。

## Key Decisions

- 本轮保留 `d3-force`，只迁移 `RelationshipGraph`，避免破坏 `ChapterGraph`。
- 使用带命名空间的图节点 ID：`character:${name}`、`location:${name}`、`organization:${name}` 等，避免多类型同名串节点。
- `RelationshipLink.source/target` 统一改为 graph node id，不再使用角色名。
- `relationshipChanges` 保持旧格式角色名输入，在解析时映射到 `character:${name}`。
- plain data 与 graphology 分层：服务层返回 `RelationshipGraph { nodes, links, insights }`，UI/服务内部临时派生 graphology Graph。
- `relationshipStore.ts` 不重复实现 `computeInsights`，洞察统一放在 `src/lib/graph-insights.ts`。
- 重新生成功能新增 `src/services/snapshotRegeneration.ts`，不引用不存在的 `regenerateAllSnapshots`。
