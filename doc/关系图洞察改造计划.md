# 关系图洞察改造计划

## 背景

当前关系图洞察由 `computeGraphInsights(graph)` 在加载关系图时实时计算，包含“惊喜连接、孤立节点、稀疏社区、桥梁节点”等类型。

现有问题：

- 洞察没有稳定 ID，刷新前后无法判断是否为同一条。
- 洞察数量被 `slice` 截断，消失不一定代表问题已解决。
- 问题型洞察和机会型洞察混在一起，优先级不清晰。
- 洞察没有严重程度、状态、过期提示。
- 右侧栏同时承载节点详情和洞察列表，章节增多后不适合继续堆叠。
- 点击多节点洞察时只聚焦第一个节点，无法表达整组关系。

## 目标

第一阶段目标是把洞察从“临时提示列表”升级为“可分页、可分类、可聚焦的图谱分析面板”。

不做：

- 不做手工“完成”状态。
- 不做长期审计历史。
- 不做用户自定义规则。
- 不改变关系图主数据生成逻辑。

## 方案

### 1. 洞察数据模型

为每条洞察补充稳定字段：

```ts
interface InsightItem {
  id: string
  type: 'surprising-connection' | 'isolated-node' | 'sparse-community' | 'bridge-node'
  category: 'issue' | 'opportunity'
  severity: 'high' | 'medium' | 'low'
  title: string
  description: string
  nodeIds: string[]
  suggestion: string
  stale?: boolean
}
```

ID 生成建议：

- 单节点：`${type}:${nodeId}`
- 双节点：`${type}:${sortedNodeA}:${sortedNodeB}`
- 多节点社区：`${type}:${community}:${sortedNodeIdsHash}`

### 2. 洞察分类

问题类：

- `isolated-node`：孤立节点
- `sparse-community`：稀疏社区

机会类：

- `surprising-connection`：惊喜连接
- `bridge-node`：桥梁节点

默认筛选显示问题类，避免用户把所有机会提示都当成必须修复项。

### 3. 严重程度

初始规则：

- `high`：主角/主要角色孤立，或涉及过期快照的关键节点。
- `medium`：普通角色孤立、稀疏社区。
- `low`：地点、物品、事件孤立；桥梁节点；惊喜连接。

后续可根据节点类型、连接数、出场次数、关系权重继续细化。

### 4. 取消硬截断

计算层不再只返回前 16 条。

改为：

- 生成全部洞察。
- 按严重程度、类型、节点重要度排序。
- UI 使用现有 `<Pagination>` 分页展示。

这样洞察消失才更接近“规则不再命中”，不会因为被列表上限挤掉而误导用户。

### 5. 右侧栏结构

右侧栏顶部增加分段切换：

- `详情`
- `洞察`

详情页：

- 工具栏
- 图例
- 节点详情

洞察页：

- 工具栏保留必要入口。
- 筛选：待处理 / 机会 / 全部。
- 类型筛选：惊喜连接 / 孤立节点 / 稀疏社区 / 桥梁节点。
- 洞察列表。
- 分页。

点击洞察后左侧图谱即时响应，右侧仍停留在洞察列表。

### 6. 点击行为

单节点洞察：

- 聚焦该节点。

双节点洞察：

- 聚焦两端节点和它们之间的边。

多节点洞察：

- 临时显示相关子图，或高亮整组节点。

第一阶段可以先实现“设置 focusedNodeId + 保留 nodeIds”，后续再增强成多节点高亮。

### 7. 快照过期联动

如果快照重建状态中存在“已过期”章节，洞察页顶部显示提示：

```text
部分洞察可能基于过期快照，建议先重建相关章节。
```

第一阶段不要求精确映射每条洞察到章节，但应给出全局提醒。

后续如果节点能追踪来源章节，再把 `stale` 标到具体洞察上。

## 实施步骤

1. 扩展 `InsightItem` 类型，加入 `id/category/severity/stale`。
2. 拆分洞察生成规则，给每条洞察生成稳定 ID 和分类。
3. 移除洞察生成层的 `slice` 截断，改为排序后全量返回。
4. 在 `RelationshipGraph` 右侧栏增加 `详情 / 洞察` 分段。
5. 将洞察列表迁移到独立侧栏页，接入 `<Pagination>`。
6. 调整 `focusInsight`，为多节点洞察预留高亮能力。
7. 接入快照状态的全局过期提示。
8. 构建验证，并重点检查右侧栏滚动和图谱聚焦交互。

## 验收标准

- 洞察列表支持分页，不再因为数量增长挤压节点详情。
- 默认只展示问题类洞察，机会类可切换查看。
- 每条洞察有稳定 ID，刷新后同一问题 ID 不变。
- 洞察不再因硬截断而静默消失。
- 点击洞察时左侧图谱仍可见，并能聚焦相关节点。
- 存在过期快照时，洞察页有明确提示。
