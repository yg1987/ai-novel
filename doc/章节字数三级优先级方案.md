# 章节字数三级优先级方案

## 概述

为 AI 章节生成添加三级字数优先级系统，让用户可以在不同层级设定每章的预计字数。

## 优先级链

```
Editor 工具栏「预计字数」    ← 粘性手动覆盖，最高优先 (Priority 2)
        ↓ 没设
Outline 面板「预计字数」      ← 规划时填写，每章独立 (Priority 1)
        ↓ 没填
系统设置「默认预计字数」      ← 全局兜底，默认 4000 (Priority 3)
```

> 注：项目级 `target_words`（当前 100000）保持为**全书总字数目标**，用于统计面板进度条，不再参与单章生成的字数决议。

---

## 数据存储设计

### 1. 系统设置：`<workspace>/settings.json`

```json
{
  "defaultWordCount": 4000
}
```

### 2. 大纲每章元数据：`<project>/outline/_chapter_meta.json`

```json
{
  "ch001": { "expectedWords": 3000 },
  "ch002": { "expectedWords": 5000 }
}
```

### 3. Editor 手动覆盖：`<project>/memory/_chapter_wordcounts.json`

```json
{
  "ch001": 2500,
  "ch003": 4000
}
```

## 实施计划

### Step 1：系统设置 Modal

**文件**：
- 新建 `src/components/SettingsModal.tsx`
- 修改 `src/App.tsx`
- 新建 `src/services/settings.ts`

**内容**：

1. `src/services/settings.ts` — 系统设置读写
   - `loadSettings(): Promise<Settings>` — 从 `<workspace>/settings.json` 读取，不存在则返回默认值 `{ defaultWordCount: 4000 }`
   - `saveSettings(settings: Settings): Promise<void>` — 保存

2. `src/components/SettingsModal.tsx` — 设置弹窗组件
   - Props: `{ visible: boolean; onClose: () => void }`
   - 内容：`默认预计字数` 输入框（number input，step=100，min=500）
   - 初始值来自 `loadSettings()`
   - 保存按钮调用 `saveSettings()`，取消恢复原值
   - 后续可扩展其他设置项

3. `src/App.tsx` — 顶栏改造
   - 在 `⚙ AI配置` 按钮右侧添加 `🛠 系统设置` 按钮
   - 点击打开 `SettingsModal`

**预期结果**：用户可以通过顶栏按钮打开系统设置弹窗，修改全局默认预计字数并保存

---

### Step 2：Outline 面板「预计字数」字段

**文件**：修改 `src/components/OutlinePanel.tsx`

**内容**：

1. 在 Outline 编辑器区域顶部（Markdown 编辑器上方），新增一行：
   ```
   预计字数: [____] 字
   ```
   - 仅在编辑**章节细纲**时显示（`activeType === 'chapter'`）
   - 总纲和分卷大纲不显示

2. 加载逻辑：
   - 切换章节时，从 `outline/_chapter_meta.json` 读取该章的 `expectedWords`
   - 如果不存在，默认值为系统设置的 `defaultWordCount`（4000）
   - 显示在输入框中

3. 保存逻辑：
   - 用户修改后自动保存（debounce 1s），与现有 Markdown 自动保存逻辑一致
   - 如果值等于系统默认值，可以选择不写入（保持文件精简）

4. 底部添加提示文字：
   ```
   💡 AI 生成时将使用此预计字数目标
   ```

**预期结果**：用户在大纲面板编辑每章细纲时，可以设置该章的预计字数，自动保存

---

### Step 3：Editor 工具栏「预计字数」输入框

**文件**：修改 `src/components/Editor.tsx`

**内容**：

1. 在工具栏「✨ 生成」按钮左侧，新增输入框组：
   ```
   [预计字数: 3000 ↩] [✨ 生成] [保存]
   ```

2. 输入框行为：
   - 显示当前**三级优先级决议后的有效字数**
   - `type="number"`，`step=100`，`min=500`，宽度约 80px
   - 用户修改后自动保存到 `memory/_chapter_wordcounts.json`

3. 来源标注：
   - 输入框右侧小字 tooltip 显示来源
   - 📝 手动（Priority 2）、📋 大纲（Priority 1）、⚙ 系统（Priority 3）
   - 当用户手动修改后，来源变为「📝 手动」

4. 重置按钮 `↩`：
   - 仅在当前来源为「📝 手动」时显示
   - 点击后：删除 `memory/_chapter_wordcounts.json` 中该章记录，回退到下一级
   - 输入框值更新为下一级决议值

5. 决议函数新增：
   - `resolveChapterWordCount(projectId, chapterId): Promise<{ value: number; source: string }>`
   - 按优先级依次检查：手动覆盖 → 大纲元数据 → 系统设置 → 硬编码 4000

**预期结果**：用户在 Editor 中可以查看和手动覆盖每章的预计字数，有明确的来源提示

---

### Step 4：`buildContext` 改造

**文件**：修改 `src/contextEngine/index.ts`

**内容**：

1. 函数签名变更：
   ```typescript
   // 旧：接收 projectTargetWords
   buildContext(projectId, volume, chapterId, targetWords)
   
   // 新：改为内部决议，不再依赖外部传入 project.target_words
   buildContext(projectId, volume, chapterId)
   ```

2. 新增字数决议逻辑：
   ```typescript
   const { value: targetWords } = await resolveChapterWordCount(projectId, chapterId)
   ```

3. Prompt 注入保持不变：
   ```
   ## 字数要求
   本章目标字数约 N 字
   ```

4. 移除对 `project.target_words` 的依赖

**预期结果**：AI 生成时使用的字数目标来自三级优先级决议，而非项目级总字数

---

### Step 5：API 层辅助函数

**文件**：修改 `src/api/tauri.ts`

**内容**：

新增两个辅助函数（复用现有 `readProjectFile` / `writeProjectFile`）：

```typescript
// 大纲元数据读写
export async function loadChapterMeta(projectId: string): Promise<Record<string, { expectedWords?: number }>>
export async function saveChapterMeta(projectId: string, chapterId: string, meta: { expectedWords?: number }): Promise<void>

// Editor 手动覆盖读写
export async function loadChapterWordCountOverrides(projectId: string): Promise<Record<string, number>>
export async function saveChapterWordCountOverride(projectId: string, chapterId: string, wordCount: number): Promise<void>
export async function deleteChapterWordCountOverride(projectId: string, chapterId: string): Promise<void>
```

也可以考虑直接在 `buildContext` 和 Editor 组件内部用 `readProjectFile` / `writeProjectFile` 完成，不新增 API，更简洁。

**预期结果**：各层可以方便地读写自己的字数配置数据

## 不涉及改动的部分

| 文件 | 原因 |
|------|------|
| `CreateProjectDialog.tsx` | `target_words` 保持为全书总字数目标，不变 |
| `EditProjectDialog.tsx` | 同上 |
| `StatisticsPanel.tsx` | 仍用 `project.target_words` 做进度条分母 |
| `FocusModeOverlay.tsx` | 专注模式字数目标独立，不受影响 |
| `aiProvider.ts` | 不关心字数，只收 systemPrompt |

## 验收标准

1. 新项目创建后，点「✨ 生成」，默认预计字数为 4000（来自系统设置）
2. 在大纲面板设置某章预计字数为 3000，切换到该章点生成，目标为 3000
3. 在 Editor 工具栏将预计字数改为 2500，点生成，目标为 2500，来源显示「📝 手动」
4. 点击 `↩` 重置后，回退到大纲设置的 3000
5. 修改系统默认字数为 5000，新章默认值为 5000，已有设置的章不受影响
6. 旧章没有任何设置时，使用系统默认值（4000）
7. 统计面板进度条仍使用项目 `target_words`，不受影响
