### 1. 写作 Tab 侧栏重构 — 参考 QMAI / character-arc / MuMuAINovel

**问题**：写作 Tab 侧栏存在以下问题：
- 标题叫"章节"，但侧栏内容包含卷管理、素材库、专注模式等，标题不能概括主体
- 标题行塞了 5 个元素（标题 + 新建章节 + 版本历史 + 素材库 + 专注模式），挤在一起
- 分卷下拉框旁边的"新建"是灰色文字，没有点击事件，无法创建新卷
- 硬编码了默认"卷1"但无法改名，与用户自己建的卷行为不一致
- 章节列表项只有纯文字，没有操作入口（版本历史、删除等）
- 新建章节的卷归属不明确

**改进**：参考 QMAI、character-arc、MuMuAINovel 三个项目的侧栏设计，全面重排：

**侧栏结构**（从上到下）：
- **标题行**：显示书名（`projectName`），不再叫"章节"
- **工具栏**：`+ 新建分卷` 按钮，点击自动生成"卷1""卷2"…不弹输入框，不打断操作流
- **卷列表**：可折叠卷头（▼ 展开/折叠），右侧显示章数，hover 显示 ✕ 删除按钮
- **章节项**：hover 显示 🕐（版本历史）和 ✕（删除），每卷底部 `+ 新建章节`（归属明确）
- **底部**：章数统计 + 📦 素材库 / 🎯 专注模式

**分卷管理**：
- 新建：自动递增编号，无需手动起名
- 改名：点击卷头文字 → 输入框 → 回车确认，持久化到 `memory/_volume_names.json`
- 删除：hover 出现的 ✕ 按钮 → 确认弹窗 → 删除该卷下所有章节 + 清除配置

**章节管理**：
- 自动编号"第1章""第2章"
- **标题可编辑**：编辑器顶部新增标题栏，hover 显示 ✏ 按钮 → 点击编辑 → 回车确认。侧栏同步更新（如"第1章 - 重生"），持久化到 `memory/_chapter_titles.json`
- 删除：hover 出现的 ✕ 按钮 → 确认弹窗 → `deleteProjectFile` 删文件 + 刷新列表

**设计原则**（来自参考项目实践）：
- 不懂起名不打断：新建卷/章节全自动编号，事后想改再改
- 操作归属明确：每个新建章节按钮明确归属所在卷，展开才显示
- hover 显隐：操作按钮默认隐藏，减少视觉噪音
- 软引导：无卷时显示"暂无分卷，点击 + 新建分卷 开始"

**涉及文件**：`src/components/ChapterManager.tsx`（重写侧栏）、`src/components/ProjectView.tsx`（传 projectName）、`src/style.css`（新增约 180 行样式）

### 10. 世界观/角色编辑 — Markdown 符号暴露给普通用户

**问题**：世界观各条目（世界背景、力量体系等）和角色卡的编辑器中直接显示了 Markdown 的 `#` / `##` 标题符号（如 `# 世界背景\n\n## 世界概况`），普通用户看不懂这些符号的含义，也不知道该填什么内容。角色的 placeholder 同样包含 `## 角色卡` 等 Markdown 语法。

**改进**：
- 世界观中有固定子结构的条目（世界背景→世界概况/历史事件/特殊规则、势力组织→势力列表、力量体系→境界划分）拆分为独立的标签+输入框，用户只填内容，`#` 标题由代码自动拼接
- 每个子字段增加 💡 提示文字，说明这个字段应该写什么（如"修炼境界的等级名称和特征，从低到高排列"）
- 编辑区域顶部增加总体说明（如"描述这个世界的基本设定，让读者对故事发生的世界有个大致印象"）
- 自由格式条目（重要地点、全局时间线）改用人类语言作为 placeholder，不再显示 `#` 符号
- 角色卡 placeholder 从 `## 角色卡\n\nname: xxx` 改为 `角色：xxx\n身份/职业：\n外貌特征：\n...\n💡 每行填一项就行`
- 预览模式空状态提示更友好，显示可填写的子字段名称

**涉及文件**：`src/components/WorldviewPanel.tsx`、`src/components/CharacterPanel.tsx`、`src/style.css`

### 22. AI 辅助 — 角色/大纲/世界观生成统一注入世界观上下文

**问题**：角色、大纲、世界观三个面板的 AI 辅助生成只读了 `project.json`（书名 + 类型 + 简介），没有参考用户已写好的世界观条目。导致：
- 角色卡生成时不知道世界背景、力量体系等信息，设定可能与世界观冲突
- 大纲生成时不知道已有的世界观规则
- 世界观面板自己的 AI 生成也不读已有条目，每次生成是"孤立的"

**改进**：
- 新建 `src/services/aiContext.ts` — 共享工具函数 `buildAIContext(projectId)`
- 按 fallback 链组装上下文：
  1. 读 `worldview/*.md` 前 5 个文件（每文件 800 字）→ 有世界观就注入「世界观设定」
  2. 读 `project.json` → 注入「小说名称」「类型」「简介」（简介可选）
  3. 世界观/简介为空时自动跳过，至少保证有书名 + 类型
- 三个面板的 AI 生成统一替换内联 `project.json` 读取为 `buildAIContext()` 调用

**涉及文件**：`src/services/aiContext.ts`（新建）、`src/components/CharacterPanel.tsx`、`src/components/OutlinePanel.tsx`、`src/components/WorldviewPanel.tsx`
