> **记录规则**：
> 1. **日期新的放最前面**：`## YYYY-MM-DD` 降序排列
> 2. **同天内序号连续**：`### 1.` `### 2.` `### 3.` ... 从 1 开始，不重号、不跳号，有新条目直接往后追加序号
> 3. **每条必须包含**：`**问题**` → `**改进**` → `**涉及文件**` 三段式
> 4. **涉及文件必须列出具体路径**，新建文件标注「（新建）」

### 8. AI 生成输出 markdown 格式问题（根因修复）

**问题**：AI 生成的内容全是 markdown（带 `# 标题`、`\n\n` 段落分隔），直接塞进 TipTap 的 `<p>` 标签里，保存和加载后段落格式丢失。日志验证：保存时 3942 字符带段落分隔，卸载时变 3818 字符（差 124 字符 ≈ `</p><p>` 标签长度丢失）。

**根因**：
- system prompt 没约束输出格式，LLM 默认输出 markdown
- `onToken` 直接把裸文本 `insertContentAt` 进一个 `<p>`，不拆段落

**改进**：
- **删重复扩展**：`StarterKit` 已含 `Underline`，去掉冗余的单独导入（消除 `Duplicate extension names` 警告）
- **改 prompt**：`budget.ts` 新增「输出格式要求」段，明确禁止 markdown 语法（# ## ** * > -），要求纯文本 + 空行分隔段落
- **流式拆段落**：`handleGenerate` 用 `paragraphBuf` 累积 token，检测 `\n\n` 后拆成独立 `<p>` 插入
- **完成后清理**：`stripMarkdownHeadings` 扫所有 paragraph 节点，去掉开头残留的 `#`、`##`、`-` 前缀

**涉及文件**：`src/components/Editor.tsx`、`src/contextEngine/budget.ts`

### 9. 项目存档导出/导入 archive 结构修复 + 预存 TS 错误修复

**问题 1 — 导出 archive 扁平、导入扫描失败**：`archive_project` 用 `append_dir_all(".", &dir)` 打包，archive 内部文件平铺在根（`project.json`、`chapters/...`）。而 `import_project` 按子目录扫描（`entry.path().join("project.json")`），解压后 `project.json` 是文件而非目录，扫描落空 → 报「未找到有效的项目文件」。

**改进 1**：导出时用 `dir.file_name()` 取项目目录名做 archive 前缀，`append_dir_all(&dir_name, &dir)`，确保解压后文件在正确子目录（如 `b4231943-b767-4ae3-8666-e3ca57d1bf64/project.json`）。

**问题 2 — 重复导入报错**：导入已存在于书架的项目时，`loaded_id` 只在 `!projects.iter().any(...)` 分支内赋值，已存在的项目不会被设置 → `loaded_id` 保持 None → 同样报「未找到有效的项目文件」。

**改进 2**：`loaded_id = Some(meta.id)` 移到 `if` 块外面，无论项目是否已在索引中都正确返回 ID。同时新增逻辑自动清理旧版 flat-archive 导入留下的污染文件（`projects/` 下平铺的 `project.json`、`chapters/`、`worldview/` 等）。

**问题 3 — 8 个预存 TS 错误阻塞构建**：`CharacterPanel.tsx`（`moved` 可能 undefined）、`ReviewRulesEditor.tsx`（computed property 类型不匹配 + 重复 `color` 属性）、`SettingsModal.tsx`（`AppSettings` 缺 index signature）、`WorldviewPanel.tsx`（未使用变量）、`worldviewConfig.ts`（undefined 类型不兼容）导致 `tsc --noEmit` 失败，阻塞 Tauri Rust 后端编译。
