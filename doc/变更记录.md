### 6. 素材库第三阶段首批功能：TXT/EPUB 资料源与补充来源

**问题**：现有结构化素材库只能保存普通文字素材，无法把整本 TXT/EPUB 小说按章节导入、阅读、检索和摘录；网页内容受前端跨域限制，Markdown 文件和图片参考也缺少明确的导入入口与来源追溯。长文本若直接写入单条素材正文会放大内存、搜索和写作上下文成本，同时 EPUB 归档还存在路径穿越、解压膨胀和异常 XHTML 等不可信输入风险。

**改进**：
- 完成 `html5ever` 临时 API 探针并实际加入 `html5ever`、`markup5ever_rcdom`、`quick-xml`、`zip`、`encoding_rs`、`reqwest` 和 `url`；使用 ZIP + XML/XHTML 主方案解析 EPUB，不引入 MOBI fallback。
- 新增独立资料源模型与存储：整本 TXT/EPUB 保存书名、作者、格式、项目作用域、原始附件和章节顺序，章节正文按文件独立落盘；后端提供导入预览、正式导入、分页列表、详情、章节读取、章节级关键词搜索和资料源删除命令。
- TXT 支持 UTF-8、带 BOM 的 UTF-16LE/BE 和 GBK 回退解码，并提供基础章节标题识别；EPUB 支持 container、OPF 元数据、NAV 目录、spine 阅读顺序和 XHTML 纯文本提取，同时限制归档条目数量、单条目大小、总解压大小和归档路径。
- 素材 Tab 新增“资料源”工作区，复用通用 `Pagination` 和 `Modal`，支持 TXT/EPUB 预览确认、目录阅读、书内搜索、项目/全局作用域，以及从章节选文创建普通素材；摘录保存资料源 ID、章节 ID 和章节标题引用，后续继续复用第二阶段的插入、使用记录和 AI 上下文流程。
- 新增受限网页摘录：后端仅允许 HTTP/HTTPS，设置 15 秒超时、最多 5 次重定向和 5MB 响应上限，只处理 HTML/纯文本；抓取结果进入确认界面，保存时保留原始 URL 和来源站点。

### 8. AI 生成输出 markdown 格式问题（根因修复）
