# 搜索模块设计说明 v1.0

> 场景定位：**写作中查设定** — 用户写到一半需要快速查找角色/设定/章节内容，搜完继续写。

---

## 一、交互流程

```
正在写作 → 点搜索 Tab → 输入关键词 → 回车 → 浏览结果 → 点击跳转 → 回写作继续
```

- 主要交互：回车搜索。备选：点击"搜索"按钮。
- 搜索结果不显示内部排序分数（RRF score 等），用户不需要看到。
- 输入框清空后显示"输入关键词开始搜索"占位提示。

---

## 二、搜索范围（左侧侧栏）

侧栏宽度 220px（与项目其他 panel 一致），列出以下来源：

```
全部（默认）
角色
世界观
章节
备注
大纲
记忆
素材
```

底部保留"向量搜索"复选框，默认勾选。

### 来源过滤规则

选择某个来源后，**三层过滤协同工作**：

1. **后端过滤**（搜索结果时生效）：关键词搜索按目录过滤，向量搜索按 `page_id` 前缀过滤，素材搜索按来源跳过。
2. **前端过滤**（搜索结果展示时生效）：切换来源时**客户端立即过滤**已返回的结果，无需重新搜索——`allResults` 保留完整结果，`filteredResults = allResults.filter(r.source == 选中来源)`，支持"全部→角色→全部"无丢失切换。
3. **新的搜索**走后端过滤（`sources` 参数传给 `searchProjectFiles` 和 `searchResourceFiles`），避免不必要的数据传输。

### 来源过滤详情

| 用户选择 | 关键词搜索（后端） | 向量搜索（后端 → 前端过滤） | 素材搜索（跳过逻辑） |
|---------|------------------|--------------------------|-------------------|
| 全部 | 所有目录 | 不过滤 | 执行 |
| 角色 | characters/ 目录 | page_id 前缀 characters/ | 跳过 |
| 世界观 | worldview/ 目录 | page_id 前缀 worldview/ | 跳过 |
| 章节 | chapters/ 目录 | page_id 前缀 chapters/ | 跳过 |
| 备注 | notes/ 目录 | page_id 前缀 notes/ | 跳过 |
| 大纲 | outline/ 目录 | page_id 前缀 outline/ | 跳过 |
| 记忆 | memory/ 目录 | page_id 前缀 memory/ | 跳过 |
| 素材 | 跳过 | page_id 前缀 resources/ | 执行 |

**修正的 bug**：之前选了"世界观"，切换来源只改了 `sourceFilter` state，`results` 数组未做客户端过滤，用户看到的结果不变。修复后 `allResults` 保留完整数据，`filteredResults` 按来源实时派生。

## 三、搜索结果卡片

```
┌────────────────────────────────────────────────────┐
│ [角色]  林修崖                                     │ ← 来源标签 + 文件名（命中词高亮）
│                                                    │
│ …林修崖，青玄宗内门弟子，筑基后期修为…                │ ← snippet，最多三行（命中词高亮）
│                                                    │
│ characters/林修崖.md                                │ ← 灰色等宽小字
└────────────────────────────────────────────────────┘
```

- 使用专用 CSS（`search-result-item`），不再复用伏笔模块样式
- 命中词高亮：黄色背景 + 黑色文字
- 卡片 hover 变色，可点击
- 点击跳转到对应编辑面板

### 三种状态

| 状态 | 展示 |
|------|------|
| 还没搜过 | 灰色文字「输入关键词开始搜索」 |
| 搜了，有结果 | 结果卡片列表 + 分页 |
| 搜了，无结果 | 「未找到匹配"xxx"的结果」 |

**不显示错误横幅。** 这是本地文件搜索，读文件失败/API 失败一律静默降级，只写 console.error，不打扰用户。

---

## 四、分页

复用通用组件 `<Pagination>`。每页默认 15 条，可切换 15/30/50。翻页时列表区滚回顶部。

---

## 五、搜索算法

维持三路融合架构：

```
用户输入
  → CJK 分词（标点切分 + 停用词过滤 + 二元分词扩展）
    → 关键词搜索（Rust 文件系统遍历 + 子串匹配评分）
    → 向量搜索（embedding API → LanceDB ANN → 距离转相似度）
    → 素材搜索（工作区 resources/ 文件遍历）
  → RRF 融合（K=60，三路 rank 加权求和 → 排序 → 截断）
  → 返回结果
```

**修正的 bug**：向量搜索结果如果独有（关键词和素材都没命中），之前会用空字符串填充文件名和 snippet。修复为从向量返回的 `ChunkSearchResult` 中取 `heading_path` 作为文件名、`chunk_text` 作为片段。

---

## 六、后端修改

### 递归目录遍历

搜索目录时递归扫描子目录，不再只查一级目录内的直接文件。

### 文件保护

- 单个文件超过 5MB 跳过
- 隐藏文件（`.` 开头）跳过

纯文字场景下这些保护不会触发，但防止用户误放二进制文件导致崩溃。

---

## 七、前端新增

### 组件

- `HighlightText.tsx`：搜索高亮组件，用与搜索同款的分词器做匹配，保证 CJK 二元分词命中一致

### CSS

- `.search-result-item` — 结果卡片
- `.search-result-header` — 卡片头部
- `.search-result-filename` — 文件名
- `.search-result-snippet` — 片段（三行截断）
- `.search-result-path` — 路径（等宽灰色）
- `.search-highlight` — 命中高亮（黄底黑字）
- `.search-history-tag` — 搜索历史标签

---

## 八、搜索历史（可选增强）

- 按项目隔离（localStorage）
- 最多 10 条
- 搜索框下方 pills 展示，点击直接搜

---

## 九、不做的事

- ~~错误横幅~~ — 本地搜索理论上不会失败
- ~~独立搜索窗口~~ — 复杂度高、价值有限
- ~~图片搜索~~ — 素材库没有图片索引
- ~~LLM rerank~~ — 纯本地搜索不需要
- ~~并发/异步重构~~ — 纯文字 3-5MB 项目顺序遍历 <500ms
- ~~输入时自动搜索~~ — 保持显式搜索为主交互
