## 一、背景

项目中 ForeshadowPanel 有 3 个弹窗（新增/编辑、推进、删除确认），每个都手写了相同的 overlay + stopPropagation 模式。当前实现有三个问题：

### 1.1 当前实现

ForeshadowPanel 有 3 个弹窗，每个都手写了 overlay + stopPropagation。**onClick 关闭已在之前的修复中移除**，但结构仍然冗余——同一段 DOM 模式写了三遍：

```tsx
// ForeshadowPanel.tsx — 重复 3 次的模式
<div className="modal-overlay">
  <div className="modal-content foreshadow-form" onClick={(e) => e.stopPropagation()}>
    <h3>新增伏笔</h3>
    {/* ... */}
    <button onClick={() => setShowForm(false)}>取消</button>
  </div>
</div>
```

> 注：`stopPropagation` 目前已无实际作用（overlay 的 `onClick` 已被移除），但作为防御性代码保留也无害。

### 1.2 问题清单

| # | 问题 | 严重度 | 影响 |
|---|------|:---:|------|
| 1 | ~~overlay 绑了 onClick 关闭~~ | ✅ 已修复 | 之前的 fix 中已移除三处 overlay onClick |
| 2 | 模式重复 3 次 | 🟡 S2 | 同样的 overlay + content + stopPropagation 写了三遍，未来其他组件加弹窗还会再重复 |
| 3 | 无 Keyboard 支持 | 🟢 S3 | 无法按 Escape 关闭弹窗（非本次范围，后续可在 Modal 加 `onClose` prop） |

## 二、设计方案

### 2.1 单组件

```
src/components/Modal.tsx  ← 通用弹窗容器
src/style.css              ← 已有 .modal-overlay / .modal-content，无需新增
```

不拆 Hook + 组件——弹窗行为极简（打开/关闭），不需要状态 Hook。

#### `<Modal>` — 通用弹窗容器

```typescript
// src/components/Modal.tsx

import type { ReactNode } from 'react'

interface ModalProps {
  children: ReactNode
  /** 附加到 content div 的 CSS class（如 "foreshadow-form"） */
  className?: string
}

export default function Modal({ children, className }: ModalProps) {
  return (
    <div className="modal-overlay">
      <div
        className={`modal-content${className ? ` ${className}` : ''}`}
        onClick={(e) => e.stopPropagation()}
      >
        {children}
      </div>
    </div>
  )
}
```

**设计决策：**

| 决策 | 理由 |
|------|------|
| overlay **不绑定** onClick | 点击空白处不关闭弹窗，仅通过显式操作（取消/确定按钮）关闭 |
| content 统一 stopPropagation | 集中控制，不需要每个使用者自己写 |
| className 透传 | 让调用方可以附加 `foreshadow-form`、`foreshadow-advance-modal` 等样式 |
| 不内置关闭按钮 | 关闭逻辑由调用方在 children 内自行放置按钮（如"取消""确定"），保持灵活性 |
| 不内置 ESC 关闭 | 当前无需求，后续可加 `onClose?: () => void` prop |
| 单文件、零依赖 | 纯展示组件，不引入外部库 |

### 2.2 使用示例

```tsx
// 改前
{showForm && (
  <div className="modal-overlay" onClick={() => setShowForm(false)}>
    <div className="modal-content foreshadow-form" onClick={(e) => e.stopPropagation()}>
      <h3>新增伏笔</h3>
      {/* ... */}
      <button onClick={() => setShowForm(false)}>取消</button>
    </div>
  </div>
)}

// 改后
import Modal from './Modal'

{showForm && (
  <Modal className="foreshadow-form">
    <h3>新增伏笔</h3>
    {/* ... */}
    <button onClick={() => setShowForm(false)}>取消</button>
  </Modal>
)}
```

## 六、边界情况

1. **className 为空时**：不追加多余空格（条件拼接 `className ? ` ${className}` : ''`）
2. **多层弹窗**：暂不支持。若未来需要弹窗内再弹窗，可在 Modal 增加 `zIndex` prop
3. **空 children**：合法，渲染空 content div（无实际场景）
4. **动态弹窗**：由调用方 `{condition && <Modal>}` 控制显隐，Modal 本身不管理状态
